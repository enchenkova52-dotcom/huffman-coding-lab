=== ФОРМАТ ЗАКОДИРОВАННОГО ФАЙЛА (Huffman) ===

1. СЛОВАРЬ (DICTIONARY):
   - 4 байта: количество уникальных символов (int)
   - Для каждого символа:
     * 1 байт: сам символ (byte)
     * 1 байт: длина кода в битах
     * N байт: битовая последовательность кода (упакована в байты)

2. ДАННЫЕ (DATA):
   - 4 байта: количество битов в закодированных данных (int)
   - Оставшиеся байты: закодированные данные, упакованные в байты
     (последний байт может быть дополнен нулями)

3. ПРИМЕР:
   Символ 'A' (65) с кодом "101" (длина 3) -> 
   хранится как: [65, 3, 0b10100000] (биты дополнены до байта)

4. ДЕКОДИРОВАНИЕ:
   - Читаем словарь, восстанавливаем карту код->символ
   - Читаем закодированные биты
   - По битам восстанавливаем исходные данные
