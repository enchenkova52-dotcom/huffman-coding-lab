Структура закодированного файла Huffman

Файл имеет бинарный формат.

1. Magic header

Первые 4 байта файла:

HUF1

Эта часть нужна для проверки, что файл был создан данной программой.

2. Размер исходного файла

Следующие 8 байт хранят размер исходного файла в байтах.

Тип: long.

Это нужно при декодировании, чтобы программа знала, сколько байтов надо восстановить.
Также это позволяет не учитывать лишние нулевые биты в последнем байте закодированных данных.

3. Таблица частот

Далее идет таблица частот для всех возможных байтов.

Всего возможных байтов: 256.
Для каждого байта хранится частота его появления в исходном файле.
Каждая частота записывается как long, то есть занимает 8 байт.

Общий размер таблицы частот:

256 * 8 = 2048 байт.

По этой таблице при декодировании восстанавливается дерево Хаффмана.

4. Закодированные данные

После таблицы частот идут сами закодированные данные.

Каждый байт исходного файла заменяется на код Хаффмана.
Коды записываются как последовательность битов.

Так как файл записывается побайтно, последний байт может быть дополнен нулями справа.
При декодировании используется размер исходного файла, поэтому лишние нули не попадают в восстановленный файл.

Итоговая структура файла:

4 байта  - HUF1
8 байт   - размер исходного файла
2048 байт - таблица частот
остальное - закодированные данные
